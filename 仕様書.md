# 📄 仕様書：Minecraftサーバー制御システム

（Discord Bot + ConoHa for GAME API）

---

# 1. 🎯 目的

Discord上からコマンド操作で、Minecraftサーバーの起動・停止・状態確認を行う。

---

# 2. 🏗️ システム構成

```
[ユーザー]
   ↓ Discord
[Discord Bot]
   ↓ HTTPS (API)
[ConoHa API]
   ↓
[Minecraftサーバー]
```

---

# 3. 🔧 技術スタック

## 必須

* 言語：Python

  * discord.py（Bot）
  * requests（API通信）

## 任意

* dotenv（環境変数管理）
* asyncio（非同期処理）

# 4. 🔑 認証設計（ConoHa API）

## 必要情報

* テナントID
* APIユーザー名
* パスワード

## 認証フロー

1. トークン取得（POST /tokens）
2. トークンをヘッダに付与して操作

```http
X-Auth-Token: {token}
```

---

# 5. ⚙️ 機能一覧

## 5.1 サーバー起動

### コマンド

```
/start
```

### 処理

* 現在状態取得
* 停止中なら起動API呼び出し

### レスポンス

* 「起動開始しました」
* 「既に起動しています」

---

## 5.2 サーバー停止

### コマンド

```
/stop
```

### 処理

* 起動中なら停止API実行

---

## 5.3 状態確認

### コマンド

```
/status
```

### レスポンス

* ACTIVE（起動中）
* SHUTOFF（停止中）

---

## 5.4 自動停止（重要機能）

### 条件

* プレイヤー0人が一定時間継続

### 実装案

* RCON or クエリで人数取得
* 10分間0人 → 停止

---

## 5.5 起動待機通知

### 内容

* 起動完了時に通知

# 8. ⏱️ 状態管理

## サーバーステータス

| 状態      | 意味    |
| ------- | ----- |
| ACTIVE  | 起動中   |
| SHUTOFF | 停止中   |
| BUILD   | 起動処理中 |

---

# 9. 🔁 処理フロー

## 起動フロー

1. /start
2. 状態確認
3. SHUTOFFなら起動
4. ACTIVEになるまでポーリング
5. 完了通知

---

## 停止フロー

1. /stop
2. ACTIVEなら停止
3. SHUTOFF確認
4. 完了通知

---

# 10. 🚨 エラーハンドリング

| エラー    | 対応      |

| API失敗  | リトライ    |
